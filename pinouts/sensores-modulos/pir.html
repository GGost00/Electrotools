👁️ PIR HC-SR501

Sensor de Movimiento Infrarrojo Pasivo

👁️

[ESPACIO PARA IMAGEN: pir.png]

Diagrama de pinout PIR HC-SR501

📌 Pinout PIR HC-SR501 (3 pines)

Pin Etiqueta Función Conexión Arduino
1 VCC Alimentación 5V (rango: 4.5V - 20V)
2 OUT Señal de salida digital Cualquier pin digital (ej: D2)
3 GND Tierra GND Arduino

⚙️ Potenciómetros de Ajuste

Control Función Rango Notas
Sx (Sensitivity) Sensibilidad / Distancia 3m - 7m Girar horario aumenta distancia
Tx (Time) Tiempo de salida activa 0.3s - 200s (5-300s según modelo) Duración del pulso HIGH en OUT

🔧 Jumper de Modo

Posición Modo Comportamiento
H (High) Repetible / Retriggerable Cada movimiento extiende el tiempo activo. Ideal para luces que permanecen encendidas mientras hay movimiento
L (Low) No-repetible / Single Trigger Un pulso por detección, luego espera tiempo de retardo. Ideal para contar eventos

🔍 Principio de Funcionamiento

PIR = Passive Infrared (Infrarrojo Pasivo). No emite señales, solo detecta cambios en radiación infrarroja (calor).

Funcionamiento:

  1. Dos sensores piro-eléctricos detectan radiación IR (calor corporal)
  2. Lente Fresnel enfoca IR en sectores del sensor
  3. Cuando una persona se mueve, cambia el patrón IR entre sectores
  4. El chip BISS0001 procesa la señal y activa OUT a HIGH (3.3V)
  5. OUT permanece HIGH durante el tiempo configurado en Tx

⚠️ No detecta objetos estáticos, solo movimiento.

⚙️ Especificaciones Técnicas

Voltaje: 4.5V - 20V DC (típico 5V)
Corriente: <65mA en funcionamiento
Salida: HIGH = 3.3V, LOW = 0V
Distancia de detección: 3m - 7m (ajustable con Sx)
Ángulo de detección: 110° - 120° (lente Fresnel)
Tiempo de bloqueo: 2.5s después de cada detección
Tiempo de activación inicial: 30-60s (warm-up)
Temperatura operativa: -15°C a +70°C
Dimensiones: 32mm x 24mm (PCB)

🌡️ Warm-Up (Calentamiento Inicial)

⚠️ IMPORTANTE: El sensor PIR requiere 30-60 segundos de tiempo de calentamiento después de encenderse.

Durante este período:

  • El sensor estabiliza los componentes piro-eléctricos
  • Puede haber disparos falsos o lecturas erróneas
  • OUT puede cambiar de estado aleatoriamente

Solución: Ignora las primeras lecturas en tu código durante 60 segundos después del setup().

⚠️ Limitaciones y Falsos Disparos

Corrientes de aire: Cambios bruscos de temperatura (aire acondicionado, calefacción) pueden disparar el sensor.

Luz solar directa: Rayos del sol entrando por ventanas pueden causar falsas alarmas.

Mascotas: Animales pequeños (gatos, perros) también emiten IR y disparan el sensor.

Objetos calientes: Radiadores, estufas, tuberías de agua caliente pueden activar el sensor.

Solución: Ajusta sensibilidad (Sx), coloca sensor alejado de fuentes de calor, usa lentes Fresnel personalizadas.

💻 Código Arduino Básico

#define PIR_PIN 2
#define LED_PIN 13

void setup() {
  Serial.begin(9600);
  pinMode(PIR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  
  Serial.println("Calentando sensor PIR...");
  delay(60000); // Esperar 60s warm-up
  Serial.println("Sensor listo!");
}

void loop() {
  int motion = digitalRead(PIR_PIN);
  
  if (motion == HIGH) {
    digitalWrite(LED_PIN, HIGH);
    Serial.println("¡Movimiento detectado!");
  } else {
    digitalWrite(LED_PIN, LOW);
    Serial.println("Sin movimiento");
  }
  
  delay(100);
}

🔌 Compatibilidad con 3.3V (ESP32/ESP8266)

✅ El HC-SR501 puede alimentarse desde 5V hasta 20V, pero la salida OUT es 3.3V.

Conexión ESP32/ESP8266:

  • VCC del PIR → 5V (VIN del ESP32) o fuente externa
  • OUT del PIR → GPIO del ESP32 (compatible 3.3V)
  • GND del PIR → GND común

✅ No requiere divisor resistivo ni level shifter porque OUT ya es 3.3V.

  • Luces automáticas: Encender luces cuando alguien entra a una habitación
  • Sistemas de alarma: Detectar intrusos en áreas protegidas
  • Ahorro energético: Apagar equipos cuando no hay personas presentes
  • Contadores de personas: Contar personas que pasan por un punto (modo L)
  • Automatización del hogar: Activar ventiladores, música, calefacción
